좋아, 그러면 포폴용으로 최소한 **리스트 페이지 + 디테일 페이지**를 포함한 구조를 잡자.
목표: **초보도 블록 클릭으로 화면 구성 가능 + 게시판 리스트/디테일 최소 동작**

---

## 1️⃣ 폴더 구조 (게시판 포함)

```text
/src
  /components           <-- 블록 단위
    Header.ts
    Footer.ts
    Hero.ts
    Text.ts
    Image.ts
    Form.ts
    Modal.ts
    PostList.ts        <-- 게시판 리스트
    PostDetail.ts      <-- 게시판 디테일
  /pages
    Home.ts
    Posts.ts            <-- 리스트 + 디테일 컨트롤
    Editor.ts
  /state
    store.ts
    types.ts
  /utils
    dom.ts
    helpers.ts
  main.ts               <-- 진입점, 툴바 초기화, 렌더
/assets
  /images
```

* **PostList.ts**: `sections` 배열로 리스트 렌더 + 클릭 시 선택된 글 id 저장
* **PostDetail.ts**: 선택된 글 id 기반으로 상세 렌더

---

## 2️⃣ 페이지 상태 구조

```ts
type PageState = {
  sections: Section[];       // 블록 배열
  selectedId: string | null; // 선택 블록
  pagePath: "home" | "posts" | "editor";
  postSelectedId: string | null; // 게시판 선택 글
  undoStack: Section[][];
  redoStack: Section[][];
}
```

* **pagePath** → 현재 페이지 (Home / Posts / Editor)
* **postSelectedId** → 게시판에서 선택된 글 id

---

## 3️⃣ 게시판 흐름

1. **리스트 페이지 (`Posts`)**

   * PostList 컴포넌트 → 글 목록 렌더
   * 글 클릭 → `postSelectedId` 업데이트
   * 렌더 → PostDetail 표시

2. **디테일 페이지**

   * PostDetail 컴포넌트 → 선택 글 id 기반 렌더
   * 뒤로가기 / 리스트 보기 버튼 → `postSelectedId = null` → 리스트만 표시

---

## 4️⃣ 블록 단위 렌더

* 페이지 렌더 시:

```text
Header
PostList or PostDetail (조건부)
Footer
```

* 리스트/디테일 전환은 **상태 `postSelectedId`로 결정**
* 다른 블록은 독립 컴포넌트 → 자유롭게 추가/삭제 가능

---

## 5️⃣ UI 동작 흐름

| 이벤트         | 상태 변화               | 화면                          |
| -------------- | ----------------------- | ----------------------------- |
| 게시글 클릭    | `postSelectedId` = id   | PostDetail 렌더               |
| 뒤로가기       | `postSelectedId` = null | PostList 렌더                 |
| 블록 추가/삭제 | `sections` 배열 변경    | 전체 페이지 렌더              |
| Undo/Redo      | `sections` 교체         | 전체 페이지 렌더              |
| 페이지 전환    | `pagePath` 변경         | Header + 선택 페이지 + Footer |
